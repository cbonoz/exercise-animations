# Exercise Animations

Stick figure exercise animations in SVG frame sequences for [BridgeRecovery](https://github.com/cbonoz/bridge-recovery).

Animation frames are generated by an AI-friendly script — describe the exercise to an LLM and it can produce a `getFrame` function that turns stick-figure poses into smooth 3-second loops.

All exercise definitions live in [`animation-plan.json`](animation-plan.json) — structured metadata for 50+ exercises across 8 muscle groups.

## Structure

```
animation-plan.json     ← exercise catalog (all 50 exercises)
assets/animations/
  hamstring-slide/      ← one directory per exercise
  bridges/
  cat-cow/
  dead-bugs/
  clamshells/

scripts/
  generate.js           ← reads animation-plan.json, renders SVG frames
  preview.js            ← local preview server
  validate.js           ← quality checks
```

Each exercise directory contains 72 SVG frames (`frame-*.svg`) at 400×300, 24fps + an `index.html` viewer.

## Preview

```bash
npm run preview
# Opens at http://localhost:3001
```

## Generate

```bash
npm run generate
```

Produces 72 SVG frames per exercise with eased interpolation between key poses.

## Validate

```bash
npm run validate
```

Runs automated quality checks on all generated frames:

| Check | What it catches |
|-------|----------------|
| SVG validity | Unescaped `&`, mismatched tags |
| Structure | Missing/inconsistent frames |
| Labels | Labels that pop in/out of animation |
| Skeleton connectivity | Disconnected limbs, floating segments |
| Limb length consistency | Segments that unnaturally stretch/shrink |
| Cycle closure | Whether frame 71 loops back to frame 0 |
| Motion | Detect static or near-static animations |
| Smoothness | Sudden jumps between adjacent frames |
| Orientation | Figure standing when it should be lying down |
| Floor contact | Appendages going under or floating above floor |

New exercises should pass all checks before being added to the app.

## Adding a New Exercise

1. Add an entry to `animation-plan.json` with the exercise's posture, axis, and motion description
2. Add a `getFrame` function in `scripts/generate.js`
3. Run `npm run generate` to produce the frames
4. Run `npm run validate` to check for issues

The generator works by interpolating joint positions between start and end poses. The canvas is 400×300, floor line is typically at y=210–260, and each frame includes a ghost layer showing the starting pose.

This is designed for AI generation — paste an existing exercise's `getFrame` function into an LLM along with its `animation-plan.json` entry, describe the new movement, and it can produce the code.

## Usage

To consume these animations from the RN app, copy `assets/animations/*/lottie.json` into the app's asset bundle and render with `lottie-react-native`.
